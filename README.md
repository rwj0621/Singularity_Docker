为规避在 Singularity 内直接构建容器时遇到的权限与依赖问题，我们改为采用 Docker 构建镜像，再转换为 Singularity 格式以供最终使用。
# 一、创建Dockerfile
安装策略根据工具类型进行划分：依赖完整的系统工具使用 apt，版本要求较高的包使用 conda，而需要精确版本控制的特定工具则直接下载预编译二进制文件
* apt 装基础工具：
用系统自带的软件库安装那些最基础、最稳定的工具，比如编译器、系统库，保证整个环境不会出问题。
* 用 conda 装专业软件：
当我们需要更新、更专业的科研工具时，就用 conda 来找和装，这样能很方便地用到最新版本。

    **可能遇到的问题：** 包依赖链涉及官方频道，安装前需要接受条款。
* 直接下载特定工具：
当某个工具用 conda 下载太慢或者老是失败时，我们就直接去官网下载现成的程序包，手动放进去，省时又省力。
